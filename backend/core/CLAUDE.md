# Core — Django Project Configuration

## What This Is

`core/` is the Django project directory — the top-level configuration package generated by `django-admin startproject`. It wires together the Django framework, apps, middleware, database, and URL routing.

## Files

| File | Purpose |
|------|---------|
| `settings.py` | All Django settings (installed apps, middleware, database, auth, CORS, DRF config) |
| `urls.py` | Root URL config — routes `/admin/` to Django admin and `/api/` to the `api` app |
| `wsgi.py` | WSGI entry point for production deployment |
| `asgi.py` | ASGI entry point for async/production deployment |
| `__init__.py` | Package marker (empty) |

## Key Connections

- `settings.py` sets `INSTALLED_APPS` to include `'api'` and third-party apps (`rest_framework`, `corsheaders`)
- `settings.py` sets `ROOT_URLCONF = 'core.urls'` and `AUTH_USER_MODEL = 'api.User'`
- `urls.py` includes `api.urls` at the `/api/` prefix
- `wsgi.py` / `asgi.py` expose `core.urls` via the WSGI/ASGI application interface
- Static files served from `STATIC_URL='static/'`, media from `MEDIA_URL='/media/'` (dev only)

## Dependencies

- `rest_framework` — Django REST Framework for API views
- `corsheaders` — Cross-Origin Resource Sharing middleware

## When Modifying

- **settings.py**: Only edit outside of auto-generated sections when adding new apps, changing DB, or modifying middleware order
- **urls.py**: Add new top-level routes here (e.g., `path('api/v2/', include('api_v2.urls'))`)
- **Never** edit `wsgi.py` / `asgi.py` unless switching to a different server (Gunicorn, Uvicorn, etc.)

## Relationship to `api/`

```
Request → core/urls.py → api/urls.py → api/views.py
                                        → api/serializers.py
                                        → api/models.py → SQLite (db.sqlite3)
```
